IPC Protocol Wiretapping & Packet Analysis

Role

Engineer Intern

Organisation

A*STAR Advanced Remanufacturing and Technology Centre (ARTC)

Period

Sep 2022 – Apr 2023

Tech Stack

Python, Wireshark, Matplotlib, Ethernet Powerlink (EPL)

The Challenge: In-Situ Spectroscopy

To evaluate the elemental composition of metal powder during a live print, we needed to integrate a Compact Spectrometer-Firefly4000 to capture plasma emissions excited by the printer's laser.

However, the spectrometer must only collect data when the building platform and recoater blade are completely stationary to ensure accurate readings. Because commercial 3D printers are closed "black-box" systems, we had no direct way to know when the blade stopped moving. We had to reverse-engineer the machine's internal communication states to synchronize our external sensors.

Network Wiretapping & Data Capture

Working alongside senior engineers, we bypassed the software restrictions by hardware tapping the machine's Industrial PC (IPC) to eavesdrop on the internal Ethernet Powerlink (EPL) V2 network communications.

  • Utilized Wireshark to capture and filter network traffic between the managing IPC and internal controlled components (servomotors, stepper motors).
  • Monitored a live titanium print job over 30.6 hours, capturing 337 files containing over 168 million individual network packets.

Python Data Pipeline & Optimization

With the raw network traffic captured, I built a data pipeline to parse the hex strings into usable integer formats and plot them over time to uncover visual patterns corresponding to physical machine movements.

  • Developed Python scripts utilizing Pandas and Matplotlib to iteratively generate 3x4 grid graphs of 12 different packet nodes plotted against print time.
  • Optimized memory and processing loads by utilizing the iloc[::2] function to down-sample the massive dataset to alternate rows (8,279 rows). This provided the perfect balance—saving significant processing power while maintaining the distinct saw-tooth fidelity of the movement graphs.

Data Pipeline: Python script utilizing Matplotlib subplots to map 12 extracted packet nodes against epoch time.

Data Visualization: The resulting 3x4 grid revealing distinct cyclical saw-tooth patterns in the network traffic.

Reverse-Engineering Results

To decode what these mysterious packet graphs represented, I cross-referenced their visual peaks and troughs against our known InfluxDB telemetry data (which tracked actual recoater speed, dispenser position, and filter pressure).

Through this analysis, I successfully proved that "Node ID 2" perfectly correlated with the position and speed of the recoater blade. Deciphering this specific packet established the crucial baseline needed to automatically trigger the Firefly4000 spectrometer exactly when the machine was in the correct, stationary state.
